Views Field View

Vložil/a zperina, 29 Prosinec, 2013 - 14:22
Kategorie: Jak na to?
Týká se verze: Drupal 7.x

Zdravím. Modul Views Field View mě zaujal, protože by pravděpodobně mohl dělat to , co potřebuji. To jest, zobrazení cca 3 pohledu v jejich nadefinovaném tvaru s využitím společných parametrů ve filtru. Otázka. Podařilo se někomu tento modul rozchodit a jak přesně se předávají argumenty (contextual filters) z matky (hlavní views) na dítě (vložený views) ? Vložil bych views do patičky, protože zachování formátování se mi hodí, nejsou stejné počty sloupců ani datová pole. Jen některé hodnoty jsou společné. Ale nové Views se vztahy se mi nehodí. Druhá otázka, nebo spíše prosba. Používání Contextual filters ve Views jsou pro mě neustále „černá díra“. Jak mají fungovat a k čemu slouží vím, ale jak to rozchodit je opravdu problém. To fakt neexistuje žádný rozumný návod s vysvětlením a příklady, na kterých by se to dalo odzkoušet. Když už něco najdu, tak se to točí kolem ID uživatele, nebo taxonomi. Ale co když chci filtrovat podle uživatelem zadaného data ? A tento filtr pomocí argumentu aplikovat rovnou do více pohledů. Za jakýkoliv link na informace, nebo nějaké funkční příklady budu vděčný. Díky.

Přeformulování dotazu

Napsal uživatel Anonymous dne 3 Leden, 2014 - 01:30.
Moc se omlouvám, ale asi nedokáži srozumitelně vyjádřit. Takže co nejjednodušeji. Podařilo se někomu zprovoznit modul Views field view. ? Byl by funkční vzorek k nahlédnutí nastavení. Nevím jaký argument nastavit do child view, aby to fungovalo. Toto je Readme tvůrce This module allows you to embed a View as a field in another View. The View in the view-field can accept argument values from other fields of the parent View, using tokens. Here's how:
  1. Before you can add a view-field to a „parent“ view, you must create a „child“ view.
  2. Add arguments to that child view. The parent view will be passing argument values to the child so that the child knows what to display. No other settings are necessary, but validators and „argument not present“ could be set.
  3. Create a „parent“ view, if not already existing.
  4. Add child view (Global:View field). The field must be toward the bottom of the field list, or at least underneath the fields that are going to be used as arguments. (E.g., „node id“ might be used as an match between the parent and child views, so put the „node id“ field before your Global:View field in the list.)
  5. Select which View and Display to use for the child data (will require doing this in 2 steps
    • the field must be saved before the display selection is available).
  6. Find which tokens are available by looking at the „Replacement patterns“ list right below the Arguments setting. Type the token replacements, in order, separated by a comma, as the arguments for that view field. These values will be passed to the child. Make sure each field you are passing as an argument is completely clean, as in: no label, no formatting, nothing that would pass into the argument other than the desired text or number.
Stále si totiž myslím, že toto je modul, který splní mé požadavky.

Dobře

Napsal uživatel Anonymous dne 3 Leden, 2014 - 12:54.
Dobře. Pokusím se nastínit ještě srozumitelněji situaci. Tajné to opravdu není. Mám 3 typy node, práce, zboží a projekty. Na vše jsou tabulkové views, kde se zobrazují nejduležitější údaje a jsou tam i agregační součty. Číslo projektu je Title uzlu (nadpis). Toto číslo je přes node reference vloženo uzlu práce i zboží. Požadavek: Na jedné stránce vidět všechna nadefinovaná views, projekt, práce, zboží, přičemž projekt si stojí nejvýše. Na něm lze aplikovat vypublikované filtry. Filtr klient, servisoval, fakturováno jsou aplikovatelné na všechna jednotlivá views. Takže až použiji filtry na nejvýše položený projekt (tam se musí číslo vypsat, protože to není seznam z node reference, ale přímo nadpis jednotlivých node) a dostanu ve výsledku jeden konkrétní projekt, tak chci pod ním zobrazit dvě views (práce a zboží), které patří k tomuto konkrétnímu projektu. Když však nevyberu konkrétní projek, ale jen třeba měsíc, nebo klienta, chci vidět všechny projekty, práci i zboží za daný měsíc, případně pro konkrétního klienta. Řešení bude jistě více, jde o to, že ne všechna jsem už někdy použil , takže seznámení se s nimi jak pracuji asi nebude otázkou 10 minut. Pro úplnost ještě popíši co nejdetalněji co mám právě rozpracováno.
  1. Views typu projekty
  2. Vystavený filtr pro začátek Nadpis, Klient, Fakturováno
  3. Do patičky přidám pohled typu práce přes toto Globální: View (Views field view)
Embed a views_field_view field in an area. This can also accept argument tokens. Vyberu View a konkrétní pohled 4. Sem -Contextual filters Use a comma (,) or forwardslash (/) separated list of each contextual filter which should be forwared to the view. See below list of available replacement tokens. Static values are also be passed to child views if they do not match a token format. You could pass static ID's or taxonomy terms in this way. E.g. 123 or „my taxonomy term“. přes nahrazovací vzorky volím pro začátek field_klient (nevím jestli raw, nebo rendered, odzkoušeno oboje a bez výsledku, patrně chybně nastaven Contextual filter u views prace. 5. Clone pohled práce , tentokrát s vazbou na projekt (to je ten co, se vložil jako dítě do patičky pohledu Projeky) 6. Do Contextual filter volím Klient (field klient) a tady končím, protože netuším jaký typ argumentu zadat, aby došlo k předání zvoleného argumentu z matky (pohled projektu). Předpokládám , že takto předám i další argumenty oddělené separátory. Je však možné, že jsem filozofii modulu Views field view pochopil úplně špatně a tak dalším řešením (mnohem pracnějším) bude vytvoření bloku prace a zboží. A pro předávání argumentu pro contextual filter přes globální filter. Toto řešení jsem již jednou použil, ale tam byl důvod, že drupal se k informaci do argumentu dostával přes účet přihlášeného uživatele. Uživatel patří do konkrétní společnosti.
